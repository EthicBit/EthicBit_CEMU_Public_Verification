## 3. The Governance Problem

AI systems evolve faster than traditional governance records.

Modern AI systems change across models, prompts, tools, workflows, datasets, rules, agents, release artifacts, dependency graphs, and runtime environments.

This creates several governance problems:

- artifacts are published without public verification;
- claims exceed available evidence;
- governance records are not machine-checkable;
- missing evidence is treated as neutral;
- blockchain anchors are confused with certification;
- GitHub publication is confused with independent validation;
- AI system changes are not governed as evidence-bound events.

EthicBit’s thesis is simple:

```text
AI governance should not depend only on trust.
It should produce evidence.
```

## 15. Public Verification Workflow

EthicBit does not ask the public to trust a claim. It exposes evidence so the claim can be checked.

Public verification workflow:

1. Inspect files in GitHub.
2. Locate manifest, receipt, or release artifact.
3. Compute canonical SHA-256.
4. Compare with receipt.
5. Compare with Ethereum mainnet transaction payload or anchored hash.
6. Review checks and verification scripts.
7. Confirm scope boundaries and non-claims.

Correct interpretation:

```text
The public can inspect evidence and compare declared hashes, receipts, and anchors.
```

Incorrect interpretation:

```text
Public inspection equals certification.
```
