## 4. What Mechanical Ethics Assurance Means

"Mechanical Ethics Assurance" means:

- **Mechanical:** Governance controls execute deterministically, not as advisory suggestions. If evidence is missing, the gate fails. If a claim exceeds verified scope, it is blocked.
- **Ethics:** The controls operationalize ethical constraints — fairness, transparency, human oversight, decision accountability — as verifiable evidence requirements, not declarations.
- **Assurance:** Claims are bound to evidence. Outputs are verifiable. Records are auditable. Outcomes are reproducible within the declared scope.

This is a **technical infrastructure** category, not an ethics philosophy. AEM-EVOLVE™ provides the infrastructure. The organization determines the ethical values it wishes to enforce.

## 11. Claim-Bound Assurance Model

Every governance claim is bound to:
- A specific artifact or evidence package
- A verified hash (AEM v1.1)
- A defined scope and target
- A closure condition
- A governance outcome

Claims that exceed verified scope receive SCOPE_LIMITED.  
Claims without supporting evidence receive FAIL_CLOSED.  
Only verified, scoped claims receive PASS.

## 12. Human-in-the-Loop as Evidence

HITL decisions are not advisory. They are evidence artifacts. When a human reviews and approves a high-risk output, that decision is:
- Recorded with a timestamp and decision identifier
- Associated with the specific output reviewed
- Stored as a verifiable HITL evidence artifact
- Subject to AEM v1.1 hash verification
- Included in the evidence package for the related gate

Human oversight becomes machine-verifiable evidence.

## 13. Fast Path as Runtime / Pre-Emission Enforcement

Fast Path provides deterministic enforcement before output emission or material action. It operates by:
1. Inheriting a signed canonical snapshot
2. Evaluating snapshot freshness (max tick elapsed)
3. Checking claim-level ceiling against requested claim
4. Verifying authorized capabilities and prohibited actions
5. Checking constitutional equivalence
6. Emitting: PASS, BLOCK, SCOPE_LIMITED, DEGRADED, NOT_VERIFIABLE, or FAIL_CLOSED

**Important:** Fast Path does not recalculate full Triple Anchor, Strong Closure, or complete AI-ME evidence per enforcement tick. It operates on inherited state.

Fast Path does not operate within full-system validation latency. It covers only the specialized pre-emission enforcement scope.
