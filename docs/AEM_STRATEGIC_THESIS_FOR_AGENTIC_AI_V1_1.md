## 1. Strategic thesis

Agentic AI requires governance that is executable, falsifiable and
materially attestable. AEM V1.1 provides that stack with eight
simultaneous properties and a reproducible public verification path.

The strategic value is not only conceptual ethics. It is operational:
claims are bounded by evidence, custody is cryptographically linked,
anchors are publicly auditable, tampering is tested, and runtime can
fail closed.

## 2. Eight properties as category boundary

AEM V1.1 should be communicated as a category with eight required
properties, not as a single script, model or product label.

1. Mechanical Ethics executable controls.
2. Verifiable constitutional state.
3. Cryptographic chain of custody.
4. Public independent anchor verification.
5. Empirical anti-tampering proof.
6. Public falsifiability in finite time.
7. Explicit third-party scope delimitation.
8. Deterministic pre-execution gating with snapshot inheritance.

This framing reduces trivial design-arounds because the category is
defined by architecture-level simultaneity, not branding.

## 3. Why property #8 is structurally strongest for Agentic AI

The pre-execution gating layer is strategically central for agentic
systems because it moves governance from post-hoc audit to
pre-execution control.

Key implications:

- Runtime decisions are gated before material action.
- Fast loops inherit constitutional validity from fresh canonical L5
  snapshot state.
- The system avoids full heavy recomputation per tick while preserving
  fail-closed posture on drift.
- Operationally, this supports scalable agentic behavior without
  relaxing evidence discipline.

Empirical basis in current artifacts:

- `results/realtime_daemon_probe_report.json` (10ms daemon probe PASS,
  409/409 ticks OK).
- `results/realtime_millisecond_guard.json` (guard OK,
  constitutional_equivalence=true, snapshot_status=PASS, L5 ceiling).
