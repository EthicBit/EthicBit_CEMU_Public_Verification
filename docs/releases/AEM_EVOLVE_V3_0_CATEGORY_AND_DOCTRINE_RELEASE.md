## 7. Evidence Before Claims™

Evidence Before Claims™ is the operating doctrine of AEM-EVOLVE™. It means:

- No claim receives PASS without verified supporting evidence
- Where evidence scope is partial, the claim receives SCOPE_LIMITED
- Where evidence is missing or fails verification, the outcome is FAIL_CLOSED
- Claims cannot be upgraded by layer precedence — only by new evidence

This doctrine applies across all layers: AEM v1.1, AEM-EVOLVE™, AI-ME Gates, Claim Boundary Engine™, Fast Path, Triple Anchor, Strong Closure.

---

## 8. PASS / SCOPE_LIMITED / FAIL_CLOSED

AEM-EVOLVE™ v3.0 uses three canonical governance outcomes:

| Outcome | Meaning |
|---|---|
| PASS | Claim is fully supported by verified evidence within declared scope |
| SCOPE_LIMITED | Claim is partially supported; scope is bounded by verified evidence |
| FAIL_CLOSED | Claim cannot be supported; gate fails closed; emission blocked |

These outcomes are not binary. SCOPE_LIMITED is a first-class outcome that permits bounded claims while preventing overclaims.

## 10. Fast Path as Specialized Enforcement Layer

Fast Path is a deterministic pre-execution enforcement layer. It is a specialized component, not a full governance system.

**v3.0 important limitation:**

> v3.0 does not claim that all governance, anchoring, closure, AI-ME evidence execution or external validation processes operate within Fast Path latency. Fast Path is a specialized runtime/pre-emission enforcement layer.

Fast Path evaluates claim-level ceiling, snapshot freshness, authorized capabilities, prohibited actions, and constitutional equivalence. It does not recalculate full Triple Anchor, Strong Closure, or AI-ME evidence per enforcement tick.
