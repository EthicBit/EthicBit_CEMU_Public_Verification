# Constitutional Control Operating Model

## Purpose

This document defines how a constitutional document becomes executable governance in this repository.

The operating rule is:

`Constitution (normative intent) -> Controls (measurable requirements) -> Execution checks (scripts/tests) -> Audit evidence (artifacts) -> Release decision`

## Sources

- Constitutional matrix: `config/constitutional_controls.v1.json`
- Control verifier: `scripts/audit/verify_constitutional_controls.sh`
- Canonical truth artifact: `artifacts/history/swarm/official_operational_status.json`
- Mixed-audience gate artifact: `results/GATE_REPORT.json`

## Enforcement semantics

- `MUST` controls are fail-closed. A failing `MUST` control blocks constitutional compliance.
- `SHOULD` controls are warning-level and do not block by default.
- Each control binds:
  - an `articleId`
  - a `controlId`
  - one executable command
  - required evidence paths
  - target audiences

## How to execute

Run from repository root:

```bash
./scripts/audit/verify_constitutional_controls.sh
```

Output:

- `results/constitutional_controls_report.json`

Exit codes:

- `0`: all `MUST` controls passed
- `1`: at least one `MUST` control failed (`FAIL_CLOSED`)
- `2`: invocation or matrix error

## Operating model by organizational function

- Legal and Regulatory: review `articleId` to `controlId` traceability and evidence paths.
- Security and Crypto: inspect controls tied to signer policy, strict official status, and key-handling protections.
- Engineering and Platform: execute controls in CI and pre-release gates.
- Executive and Mixed Audiences: consume profile outputs generated by canonical mixed-audience flow (`results/*.md`, `results/index.json`).

## Mandatory governance cycle

1. Update constitutional text (if principles changed).
2. Update `config/constitutional_controls.v1.json`.
3. Run `verify_constitutional_controls.sh`.
4. Review `results/constitutional_controls_report.json`.
5. Only publish if constitutional status is `PASS`.

## Traceability note

This model keeps narrative documents and runtime behavior aligned by design:

- no constitutional claim should exist without a mapped control;
- no mapped control should exist without executable verification and evidence output.
